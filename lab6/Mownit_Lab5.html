Postawowe informacje¶

  • R for Data Science: https://r4ds.had.co.nz/
  • R Packages (zaawansowana): https://r-pkgs.org/
  • ggplot2: https://ggplot2-book.org/

Operator %>% pozwala przekazywa wyjcie z jednej funkcji jako wejcie kolejnej, co pozwala na tworzenie acucha wywoa funkcji (analogiczne do '|' w bashu).

In [48]:
avg_dplyr <- results %>% 
    mutate(speed = n/time) %>% # tylko aby pokaza mutate, nie wykorzystujemy do plota
    group_by(n, alg) %>%
    summarise(
        time_sd = sd(time),
        time = mean(time),
        speed = mean(speed)
    )  %>% 
    ungroup() %>%
    arrange(alg, n)

Shiny¶

Shiny to biblioteka do szybkiego tworzenia interaktywnych webaplikacji/dashboardw. Przykad: https://ben18785.shinyapps.io/distribution-zoo/

Cao wymaga 2 funkcji:

  • ui - front - inputy (checkboxw, list, sliderw itp) oraz outputy (wykresy, policzone wartoci)
  • server - "backend" - na podstawie inputw generuje konkretne outputy

Przykad - zmiana stopnia aproksymacji na powyszych danych.

In [65]:
server <- function(input, output) {
  # input - lista indeksowana przez inputId z inputow z ui
  x <- c(32,64,96,118,126,144,152.5,158)  
  y <- c(99.5,104.8,108.5,100,86,64,35.3,15)
  mydata <- data.frame(x,y)
  
  output$approxPlot <- renderPlot({
    fit <- lm(y ~ poly(x, input$degree, raw=TRUE), data=mydata)
    newdata = data.frame(x = seq(input$genRange[1], input$genRange[2], length.out=250))
    newdata$y = predict(fit, newdata)
    ggplot(mydata, aes(x,y)) + geom_point() + geom_line(data=newdata)
  })
  
}
